🏗️ Architecture

Three-Phase Pipeline

A progressive journey from classical baselines to a state-of-the-art hybrid deep learning system.

Phase 1

Classical Machine Learning Baselines

The foundation phase establishes performance baselines using traditional ML classifiers on handcrafted audio features. The pipeline processes raw infant audio through preprocessing, augmentation, and a 106-dimensional feature extraction stage.

🎵 Audio Preprocessing

  • Spatial resampling to 16 kHz mono
  • Peak normalization (amplitude −1.0 to +1.0)
  • Energy-based silence trimming (25 dB threshold)

🔊 Noise Augmentation

  • Gaussian hiss injection (SNR ≈ 20 dB)
  • Pink 1/f background noise (SNR ≈ 15 dB)
  • Temporal shift (max 10%)

📐 Feature Engineering

  • MFCCs + Δ + ΔΔ (78 dimensions)
  • Chroma pitch profiles (24 dimensions)
  • Spectral centroid (2 dims) + ZCR (2 dims)
  • Total: 106-dimensional feature manifold

🤖 Classifiers

  • SVM with RBF kernel + GridSearchCV
  • Random Forest with SMOTE oversampling
  • Balanced class weights throughout
SVM Accuracy ~23%
RF Macro F1 ~31%
Phase 1 — End-to-end classical ML pipeline
Phase 2

CryNet — CNN-BiLSTM-Attention

The core innovation phase deploys CryNet (~850K parameters), a bespoke deep learning architecture built entirely from scratch in PyTorch. It processes log-mel spectrograms through spatial, temporal, and attentive pathways.

🧠 Stage 1: CNN Extractor

  • 3 cascading Conv2D blocks (1→32→64→128)
  • BatchNorm → GELU activation → MaxPool
  • Squeeze-and-Excitation (SE) channel attention
  • Residual skip connections for gradient flow

🔄 Stage 2: Temporal Encoder

  • 2-Layer Bidirectional LSTM
  • Forward/Backward: 256→128 each
  • 256-dim concatenated output
  • LayerNorm stabilisation

🎯 Stage 3: Self-Attention

  • Multi-Head Self-Attention (4 heads, d_k = 64)
  • Q, K, V linear projections
  • Scaled dot-product attention
  • Residual + LayerNorm

⚡ Training Methodology

  • Focal Loss with class-balanced weighting
  • SpecAugment regularisation (freq + time masking)
  • OneCycleLR scheduler with warmup
  • 80 epochs, early stopping on validation loss
Accuracy ~25.8%
Macro F1 ~35.4%
Phase 2 — CryNet CNN-BiLSTM-Attention architecture
Phase 3

Hybrid Fusion Architecture

The fusion phase combines the strengths of both approaches — merging learned deep representations from CryNet with handcrafted statistical features to create a unified multi-stream classifier.

🔗 Dual-Stream Fusion

  • Stream A: CryNet deep embeddings (256-dim)
  • Stream B: Handcrafted features (106-dim)
  • Concatenation-based late fusion
  • Joint representation: 362-dim vector

🎛️ Fusion Classifier

  • Fully-connected classifier head
  • Batch normalisation + dropout (0.4)
  • GELU activations throughout
  • 8-class softmax output

📈 Dynamic Ablation

  • Systematic component removal testing
  • Individual contribution measurement
  • Optimal fusion weight discovery
  • Performance vs complexity trade-off analysis

🏥 Clinical Integration

  • Grad-CAM visual explanations
  • Model interpretability dashboards
  • Per-class confidence calibration
  • Real-time inference pipeline

Multilingual Cry Corpus

Transformer-Based Architecture

Replace BiLSTM with a full Vision Transformer (ViT) or Audio Spectrogram Transformer (AST) backbone, leveraging pre-trained models and transfer learning for dramatically improved accuracy.
